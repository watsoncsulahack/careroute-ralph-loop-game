# LOCAL RALPH ROUND 03

## Top issue identified
Runs felt too static/repetitive; players needed real tradeoff pressure to make decisions feel meaningful.

## Focused improvement implemented
Introduced lightweight **Live Constraint** events per stage:
- each stage now surfaces a contextual disruption (e.g., crew shortage, policy audit)
- decisions now include event-sensitive consequences in readable feedback
- event handling can strengthen or strain cross-team momentum, adding replay variety

## Mobile-friendliness check
Events are presented as short text in the stage card; no added interaction complexity.
